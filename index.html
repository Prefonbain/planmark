Firebase (collaboration backend)

Free Firebase account is required so notes and markups sync live between everyone reviewing the plans. Paste your config values below.

How to get these values (5 min)
  1. Go to console.firebase.google.com and sign in with a Google account.
  2. Click Add project, name it anything (e.g. smith-home-review), and follow the wizard. You can skip Google Analytics.
  3. In the left menu, click Build → Firestore Database → Create database. Choose Start in test mode and pick a region close to you.
  4. Back on the project overview page, click the </> Web icon to register a web app. Give it a nickname (e.g. "Review site") — no need for Firebase Hosting.
  5. Copy the six values from the firebaseConfig snippet that appears and paste them below.
  6. See the Security Rules section at the bottom of this page before sharing with others.

Hosting your review site

GitHub Pages is the simplest free option — this very page is hosted there.

1. Put your files in a GitHub repo

Create a new public repo on GitHub. Upload:

  • The review.html file you just downloaded
  • Your floor plan images as PNGs, named to match the prefixes you configured (e.g. v1-sheet-1.png through v1-sheet-7.png)

2. Turn on GitHub Pages

Repo → Settings → Pages. Under "Build and deployment," pick source Deploy from a branch, branch main, folder / (root), then Save. A minute later GitHub gives you a URL like https://yourname.github.io/your-repo/review.html — share that with your reviewers.

3. Tighten Firebase security rules (important)

Test mode lets anyone with your project ID read and write for 30 days. Before you share the site, go to Firestore Database → Rules and paste these rules — they lock writes to the collections this tool actually uses, and are a reasonable baseline for a family-and-builder review group:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{collection}/{doc} {
      allow read: if true;
      allow write: if collection in [
        'notes', 'planHearts', 'markups',
        'photos', 'checklist'
      ];
    }
  }
}

Anyone with the URL can still read and write — there's no login. That's fine for a trusted review group. If you need stronger access control, add Firebase Authentication; this template does not include that.

4. Share the review URL

Send the URL to your builder and reviewers. When someone opens it, they pick a name from your preset list (or type a different one) — that name shows on all their notes and markups.

Updating plans later

When your builder sends a new revision, upload the new PNGs to the repo (e.g. v4-sheet-1.png...) and come back to this wizard with the new version added. Download and replace review.html. Existing notes on older versions stay — they're scoped per revision.
